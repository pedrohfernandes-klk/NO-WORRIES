# NO WORRIES

**NO WORRIES** is a one-minute audiovisual browser installation.

It is a small cinematic web piece about ordinary collapse, digital escape, work pressure, money fantasies, disappearance, and the strange possibility of return.

The experience runs as a single HTML file and combines generative visuals, animated text, ghostly search interfaces, embedded audio, synthetic voice fragments, and timed narrative cues.

## Concept

The piece follows a short emotional arc:

1. **Denial** — “no worries”, “probably fine”
2. **Erasure** — deleting, vanishing, reducing the self
3. **Body signal** — anxiety, stomach, pressure
4. **Money trap** — shortcuts, crypto, gambling, adult-content ghosts
5. **Disappearance** — the search field becomes existential
6. **Return** — not triumph, not cure, but presence

The ending does not explain the experience. It leaves an afterimage:

> STILL HERE
> what remains.

## Features

* Single-file HTML installation
* Canvas-based animated visual world
* Timed text fragments and phrase constellations
* Search-engine apparitions as narrative events
* Ghost double-search effect during the money phase
* Embedded audio beds
* Sparse synthetic voice cues
* Subtitles / spoken-text timing
* Mobile and desktop compatible
* No build process required

## How to run

Open the HTML file directly in a modern browser.

For best results:

* use Chrome, Edge, or Safari
* enable sound
* view in fullscreen
* use headphones

## File structure

```text
index.html
```

The project is intentionally self-contained. All visuals, text logic, timing, styling, and embedded audio are contained in the HTML file.

## Suggested GitHub Pages setup

1. Rename the final HTML file to `index.html`.
2. Push it to a GitHub repository.
3. Go to **Settings → Pages**.
4. Set the source to the main branch.
5. Open the generated GitHub Pages URL.

## Notes

This is not a conventional website or app. It is closer to a short browser-based installation film.

The timing, text, sound, and visual glitches are part of the experience. The piece is designed to feel unstable, but it should remain readable and playable.
